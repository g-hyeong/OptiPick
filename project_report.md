# OptiPick 프로젝트 결과보고서

## 1. 과제 개요

### 가. 과제 선정 배경 및 필요성

온라인 쇼핑 시장의 급격한 성장으로 소비자들은 수많은 상품 중에서 최적의 선택을 해야 하는 상황에 놓여 있다. 그러나 현재 온라인 쇼핑 환경에서는 다음과 같은 문제점이 존재한다:

- 정보 과부하: 각 상품 페이지마다 방대한 양의 정보가 산재되어 있어, 소비자가 핵심 정보를 파악하기 어려움
- 비교의 어려움: 여러 상품을 비교하려면 각 페이지를 오가며 수동으로 정보를 수집해야 하는 번거로움 존재
- 주관적 판단의 한계: 소비자 개인의 기준에 맞는 객관적인 비교 분석 도구 부재
- 시간 소요: 상품 조사 및 비교에 과도한 시간이 소요되어 효율적인 의사결정이 어려움

이러한 문제를 해결하기 위해 LLM(Large Language Model) 기술을 활용한 지능형 상품 비교 분석 시스템의 개발이 필요하다.

### 나. 과제 주요내용

본 과제는 Chrome Extension과 LangGraph 기반 AI Agent를 결합하여 사용자가 웹에서 발견한 상품들을 자동으로 분석하고 비교할 수 있는 시스템을 개발한다.

주요 기능:

1. 상품 정보 자동 추출
   - 쿠팡, 네이버 등 주요 쇼핑몰의 상품 페이지에서 정보 자동 추출
   - OCR 기술을 활용한 이미지 내 텍스트 정보 추출
   - LLM 기반 상품 특징, 장단점 분석

2. 지능형 상품 비교
   - 사용자가 지정한 기준에 따른 맞춤형 비교 분석
   - LLM이 추출한 객관적 비교 기준과 사용자 기준의 통합
   - 기준별 중요도 및 종합 평가 제공

3. 대화형 AI 챗봇
   - 비교 분석 결과를 기반으로 한 Q&A 기능
   - 상품에 대한 추가 질문 및 상담 지원

### 다. 최종결과물의 목표

- 사용자가 3클릭 이내에 상품 정보를 추출하고 저장할 수 있는 Chrome Extension
- 2~10개 상품을 동시에 비교 분석할 수 있는 AI 기반 비교 시스템
- 비교 결과에 대해 자연어로 질의응답이 가능한 챗봇 인터페이스
- 직관적이고 사용하기 쉬운 UI/UX

## 2. 과제 수행방법

### 가. 시스템 아키텍처 설계

본 시스템은 크게 Chrome Extension과 Agent(FastAPI) 두 부분으로 구성된다. Chrome Extension은 ExtractPage(상품 추출), ComparePage(상품 선택), CompareReportPage(결과 + 챗봇) 세 개의 주요 페이지로 구성되며, Background Worker가 장기 작업을 처리한다. Agent는 LangGraph Workflows를 통해 SummarizePage Graph, CompareProducts Graph, Chatbot Graph 세 가지 워크플로우를 운영하며, LLM(Gemini)과 OCR Service를 활용한다.

### 나. 기술 스택

Frontend: TypeScript, React, Vite, TailwindCSS
Extension: Chrome Extension API, IndexedDB (Dexie)
Backend: Python, FastAPI, LangGraph
AI/ML: Google Gemini 2.0 Flash, Clova OCR
상태관리: LangGraph MemorySaver (HITL 패턴)

### 다. LangGraph 워크플로우 설계

1. SummarizePage Graph (상품 분석)
route_node에서 시작하여 domain_parser_node 또는 parse_content_node로 분기되고, validate_page_node(LLM)를 거쳐 ocr_node에서 이미지 텍스트를 추출한 후 analyze_product_node(LLM)에서 최종 분석을 수행한다.

2. CompareProducts Graph (비교 분석, HITL)
collect_user_criteria_node에서 사용자 입력을 대기하고, analyze_products_node(LLM)에서 상품을 분석한 후 generate_report_node(LLM)에서 최종 보고서를 생성한다.

3. Chatbot Graph (대화형 Q&A)
chat_node(LLM)에서 제품 컨텍스트와 대화 히스토리를 바탕으로 응답을 생성한다.

### 라. 개발 방법론

- Agile 방식의 반복적 개발
- 컴포넌트 단위 독립 개발 및 테스트
- 도메인별 파서 모듈화 (쿠팡, 네이버 등 확장 가능)

### 나. 최종결과물 주요특징 및 설명

1) 상품 정보 자동 추출 (SummarizePage Graph)

특징:
- 도메인별 최적화된 파싱 (쿠팡, 네이버 전용 파서)
- OCR을 통한 이미지 내 스펙 정보 추출
- LLM 기반 상품 특징, 장단점 자동 분석

처리 흐름:
1. 사용자가 상품 페이지에서 Extension 클릭
2. Content Script가 페이지 HTML 추출
3. 도메인별 파서 또는 일반 파서로 콘텐츠 정제
4. OCR로 이미지 내 텍스트 추출
5. LLM이 종합 분석하여 구조화된 상품 정보 생성

2) 지능형 비교 분석 (CompareProducts Graph)

특징:
- Human-in-the-Loop(HITL) 패턴으로 사용자 기준 반영
- LLM이 자동으로 비교 가능한 기준 추출
- 기준별 중요도 점수화 (1-10점)
- 종합 평가 및 추천 제공

처리 흐름:
1. 사용자가 2~10개 상품 선택
2. 시스템이 "중요한 기준을 입력해주세요" 요청
3. 사용자가 기준 입력 (예: "배터리, 가격, 무게")
4. LLM이 상품 데이터에서 비교 기준 추출 및 분석
5. 기준별 스펙, 상세 설명, 종합평 포함 보고서 생성

3) 대화형 AI 챗봇 (Chatbot Graph)

특징:
- 비교 대상 상품 정보를 컨텍스트로 유지
- 대화 히스토리 관리로 맥락 있는 응답
- SSE 스트리밍으로 실시간 응답 표시
- 다중 세션 관리 지원

활용 예시:
- "Model A와 Model B 중 배터리가 더 좋은 건 뭐야?"
- "가성비가 좋은 제품은?"
- "이 제품의 단점을 자세히 알려줘"

4) 사용자 인터페이스

Extension 페이지 구성:

페이지 / 기능
ExtractPage / 현재 페이지 상품 추출, 카테고리 지정
ProductsPage / 저장된 상품 목록 관리, 검색, 필터링
ComparePage / 비교할 상품 선택 (2~10개)
CompareReportPage / 비교 결과 테이블 + 챗봇 사이드바

## 4. 기대효과 및 활용방안

### 가. 기대효과

1) 소비자 측면
- 시간 절약: 수동 비교 대비 80% 이상 시간 단축
- 객관적 분석: LLM 기반 편향 없는 비교 정보 제공
- 맞춤형 비교: 사용자가 중요시하는 기준에 따른 분석
- 정보 접근성 향상: 이미지 내 정보도 OCR로 추출하여 분석

2) 기술적 측면
- LangGraph 활용 사례: HITL 패턴을 활용한 실용적인 AI 워크플로우 구현
- 확장 가능한 구조: 도메인 파서 추가만으로 새로운 쇼핑몰 지원 가능
- 모듈화된 설계: 각 Graph가 독립적으로 동작하여 유지보수 용이

### 나. 활용방안

1) 일반 소비자
- 고가 전자제품 (노트북, 스마트폰 등) 구매 시 비교 분석
- 가전제품, 가구 등 스펙 비교가 필요한 상품 선택
- 리뷰 분석 및 장단점 파악

2) 전문 리뷰어/인플루언서
- 상품 리뷰 콘텐츠 제작을 위한 기초 자료 수집
- 객관적 비교 데이터 확보

3) 기업/B2B
- 경쟁사 제품 분석
- 시장 조사 및 벤치마킹
- 구매 의사결정 지원 도구

4) 확장 가능성
- 해외 쇼핑몰 지원 (Amazon, eBay 등)
- 서비스 상품 비교 (보험, 금융 상품 등)
- API 서비스화하여 타 서비스에 비교 기능 제공

## 5. 결론 및 제언

### 결론

본 과제를 통해 LLM과 LangGraph를 활용한 지능형 상품 비교 분석 시스템 "OptiPick"을 성공적으로 개발하였다.

주요 성과:
- Chrome Extension과 AI Agent의 효과적인 결합으로 사용자 친화적인 인터페이스 구현
- LangGraph의 HITL 패턴을 활용하여 사용자 맞춤형 비교 분석 실현
- OCR 기술 연동으로 이미지 기반 정보까지 분석 범위 확장
- SSE 스트리밍 기반 챗봇으로 실시간 대화형 인터페이스 제공

본 시스템은 소비자의 합리적인 구매 의사결정을 지원하고, 온라인 쇼핑 경험을 크게 개선할 수 있는 잠재력을 보유하고 있다.

### 제언

1) 단기 개선 사항
- 더 많은 쇼핑몰 도메인 파서 추가 (11번가, G마켓 등)
- 비교 결과 공유 기능 (링크, 이미지 내보내기)
- 가격 변동 알림 기능

2) 중장기 발전 방향
- 사용자 선호도 학습을 통한 개인화 추천
- 리뷰 데이터 수집 및 감성 분석 통합
- 멀티모달 LLM 활용으로 이미지 직접 분석 (OCR 대체)
- 모바일 앱 버전 개발

3) 기술적 고려사항
- 대규모 사용자 대응을 위한 인프라 확장 (VectorDB, 캐싱)
- LLM 비용 최적화 (모델 선택, 프롬프트 최적화)
- 개인정보 보호 및 데이터 보안 강화
