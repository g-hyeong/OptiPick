## 요약

온라인 쇼핑 시 소비자들은 여러 상품 페이지를 오가며 정보를 수집하고 비교하는 데 많은 시간을 소비하고 있다. 본 연구에서는 LLM과 LangGraph를 활용하여 상품 정보를 자동으로 추출하고, 사용자 기준에 따라 비교 분석하는 Chrome Extension 기반 서비스 OptiPick을 개발하였다. 기존 가격비교 서비스가 가격 중심의 단순 비교만 제공하는 한계를 극복하기 위해, OCR 기술로 이미지 내 스펙 정보까지 추출하고 Human-in-the-Loop 패턴을 적용하여 사용자 맞춤형 비교를 제공한다. 이를 통해 소비자는 상품 비교에 소요되는 시간을 크게 줄이고 합리적인 구매 의사결정을 할 수 있다.

---

## 1. 서론

온라인 쇼핑 시장은 지속적으로 성장하고 있으며, 2024년 기준 국내 온라인 쇼핑 거래액은 200조 원을 돌파하였다. 이러한 성장과 함께 소비자들은 수많은 상품 중에서 최적의 선택을 해야 하는 복잡한 의사결정 상황에 놓이게 되었다. 특히 노트북, 스마트폰과 같은 고가의 전자제품을 구매할 때는 10개 이상의 상품을 비교하는 경우가 많아 상당한 시간이 소요된다.

그러나 현재 온라인 쇼핑 환경에서는 여러 문제점이 존재한다. 첫째, 각 상품 페이지에는 상품 설명, 스펙 표, 이미지 내 텍스트, 리뷰 등 방대한 양의 정보가 비정형적으로 산재되어 있어 핵심 정보를 파악하기 어렵다. 둘째, 여러 상품을 비교하려면 각 페이지를 오가며 수동으로 정보를 수집하고 엑셀 등에 정리해야 하는 번거로움이 존재한다. 셋째, 다나와나 에누리 같은 기존 가격비교 서비스들은 가격 정보 중심의 단순 비교만 제공하며, 상품의 세부 스펙이나 장단점에 대한 심층적인 분석은 제공하지 못한다[1]. 또한 사용자 개인의 구매 기준에 맞는 맞춤형 비교 분석 도구가 부재한 실정이다.

본 연구에서는 이러한 한계를 극복하기 위해 LLM과 LangGraph를 활용한 상품 비교 분석 서비스 OptiPick을 개발하였다. 사용자가 쇼핑몰에서 상품 페이지를 방문하면 자동으로 정보를 추출하고, 원하는 상품들을 선택하여 사용자가 중요하게 생각하는 기준에 따라 비교 분석을 받을 수 있다. 또한 비교 결과에 대해 챗봇으로 추가 질문도 할 수 있어, 마치 전문 상담사에게 조언을 구하는 것과 같은 경험을 제공한다.

## 2. 관련 연구

상품 비교를 돕기 위한 다양한 연구와 서비스가 존재한다. 크롤링 기반의 가격비교 시스템[1]은 여러 쇼핑몰의 실시간 가격 정보를 수집하여 제공하지만, 상세 스펙 비교에는 한계가 있다. 사용자 리뷰 감성 분석 시스템[2]은 딥러닝을 활용하여 상품에 대한 평판을 분석하지만, 리뷰의 주관성과 신뢰성 문제가 존재한다. 또한 특정 카테고리에 대해 구조화된 스펙 비교를 제공하는 시스템도 있으나, 사전 정의된 스펙 항목에 한정되어 유연성이 부족하다.

LangGraph는 LangChain 팀에서 개발한 LLM 애플리케이션 워크플로우 프레임워크로, 복잡한 에이전트 시스템을 그래프 형태로 구성할 수 있다[3]. 주요 특징으로는 상태 기반 실행(Stateful Execution), 조건부 분기(Conditional Branching), Human-in-the-Loop(HITL) 패턴 지원이 있다. HITL 패턴은 워크플로우 중간에 사용자 입력을 받고 재개할 수 있어, 사용자와 상호작용하는 시스템 구축에 적합하다.

온라인 쇼핑몰에서는 상당량의 정보가 이미지 형태로 제공된다. 특히 상세 스펙, 제품 특징 등이 텍스트가 아닌 이미지로 표현되는 경우가 많아 OCR 기술의 활용이 필요하다[4]. 최근 딥러닝 기반 OCR 기술의 발전으로 복잡한 레이아웃의 이미지에서도 높은 정확도로 텍스트를 추출할 수 있게 되었다.

기존 연구들은 사용자가 직접 정보를 수집하거나, 사전 정의된 항목만 비교할 수 있다는 한계가 있다. 본 연구에서는 LLM을 활용하여 비정형 상품 정보를 자동으로 구조화하고, HITL 패턴으로 사용자가 원하는 기준에 따른 맞춤형 비교를 제공하는 서비스를 개발하였다.

## 3. OptiPick 서비스

### 3.1 시스템 아키텍처

<그림 1> 시스템 아키텍처

OptiPick은 Chrome Extension과 Agent(FastAPI) 두 부분으로 구성된다. Extension은 사용자 인터페이스와 웹페이지 콘텐츠 추출을 담당하고, Agent는 세 가지 LangGraph 워크플로우(SummarizePage, CompareProducts, Chatbot)를 통해 분석을 수행한다. Extension과 Agent 간의 통신은 HTTP API를 통해 이루어진다.

Extension은 네 개의 페이지로 구성된다. ExtractPage에서 현재 상품 페이지의 정보를 추출하고, ProductsPage에서 저장된 상품들을 카테고리별로 관리한다. ComparePage에서 비교할 상품들을 선택하면, CompareReportPage에서 비교 결과 테이블과 챗봇을 제공한다. 분석 작업은 수십 초가 소요될 수 있으므로 Background Worker에서 처리하여 팝업이 닫혀도 작업이 계속 실행되도록 구현하였다. 로컬 데이터 저장소로는 Chrome Storage API의 용량 제한을 극복하기 위해 IndexedDB를 사용하였다.

### 3.2 상품 정보 추출 (SummarizePage Graph)

사용자가 쿠팡, 네이버 등의 상품 페이지에서 Extension을 클릭하면 SummarizePage Graph가 실행된다. [그림 2]와 같이 route_node에서 URL의 도메인을 판별하여 주요 쇼핑몰은 전용 파서로, 일반 사이트는 범용 파서로 처리한다. 쿠팡 파서는 상품 페이지의 JSON-LD 구조화 데이터와 CSS 선택자를 활용하여 상품명, 가격, 할인율 등을 추출하고, 네이버 파서는 상품 상세 API 응답 구조를 파싱한다.

<그림 2> SummarizePage Graph 워크플로우

온라인 쇼핑몰에서는 상당량의 정보가 이미지 형태로 제공되므로, ocr_node에서 Clova OCR API를 활용하여 이미지 내 스펙 정보를 추출한다. validate_page_node에서는 LLM을 활용하여 추출된 콘텐츠가 실제 상품 페이지인지 검증하고, 검색 결과 페이지나 카테고리 페이지를 필터링한다. 최종적으로 analyze_product_node에서 LLM(Google Gemini)이 HTML 텍스트, OCR 결과, 구조화된 데이터를 종합하여 상품명, 가격, 주요 특징, 장단점을 구조화된 JSON 형태로 생성한다.

### 3.3 비교 분석 (CompareProducts Graph)

CompareProducts Graph는 Human-in-the-Loop 패턴을 적용한 비교 분석 워크플로우이다. 사용자가 2~10개의 상품을 선택하면 워크플로우가 시작되고, collect_user_criteria_node에서 interrupt_before 설정으로 중단되어 사용자에게 "중요한 비교 기준을 입력해주세요"라고 요청한다. 사용자가 "배터리, 가격, 무게"와 같이 기준을 입력하면 워크플로우가 재개된다.

analyze_products_node에서 LLM이 상품 데이터로부터 비교 가능한 모든 기준을 추출하고, 사용자가 입력한 기준과 통합한다. generate_report_node에서는 각 상품별, 기준별 스펙과 상세 설명, 기준의 중요도(1-10점), 종합평을 포함한 비교 보고서를 생성한다. 사용자가 입력한 기준은 자동으로 높은 중요도가 부여된다. 상태 관리는 LangGraph의 MemorySaver를 사용하여 thread_id별로 독립적으로 관리되어, 여러 사용자의 동시 요청을 처리할 수 있다.

### 3.4 대화형 챗봇 (Chatbot Graph)

비교 결과 페이지에서 사용자는 챗봇을 통해 추가 질문을 할 수 있다. 세션이 시작될 때 비교 대상 상품들의 정보가 컨텍스트로 저장되며, chat_node에서 LLM이 이를 참조하여 답변한다. "Model A와 B 중 배터리가 더 좋은 건?", "가성비가 좋은 제품은?", "이 제품의 단점을 자세히 알려줘" 등의 질문에 상품 정보를 기반으로 구체적인 답변을 제공한다. 대화 히스토리는 LangGraph의 add_messages reducer를 통해 자동으로 관리되어 맥락 있는 대화가 가능하다. 실시간 응답을 위해 Server-Sent Events(SSE) 스트리밍을 구현하여, 사용자는 LLM이 응답을 생성하는 동안 토큰 단위로 실시간 출력을 확인할 수 있다.

### 3.5 서비스 사용 화면

<그림 3> OptiPick 비교 결과 화면

[그림 3]은 OptiPick의 비교 결과 화면이다. 사용자가 선택한 상품들이 기준별로 비교되어 테이블 형태로 표시되며, 각 기준의 중요도와 종합평도 함께 제공된다. 우측 사이드바에서 챗봇을 통해 추가 질문을 할 수 있어, 사용자는 여러 상품 페이지를 오가지 않고도 한눈에 비교 정보를 확인하고 궁금한 점을 바로 질문할 수 있다.

## 4. 결론

본 연구에서는 LangGraph 기반의 상품 비교 분석 서비스 OptiPick을 개발하였다. 쿠팡, 네이버 등 주요 쇼핑몰에 대한 전용 파서를 구현하여 정보 추출의 정확도를 높였으며, OCR 기술로 이미지 내 정보까지 추출하였다. HITL 패턴으로 사용자 맞춤형 비교 기준을 반영하고, 챗봇을 통해 추가 질의응답을 지원하여 기존 가격비교 서비스의 한계를 극복하였다.

이를 통해 소비자는 상품 비교에 소요되는 시간을 크게 줄이고 합리적인 구매 의사결정을 할 수 있을 것으로 기대된다. 향후 더 많은 쇼핑몰 도메인 파서를 추가하고, 사용자 선호도 학습을 통한 개인화 추천 기능을 구현하면 사용자 만족도를 더욱 높일 수 있을 것이다. 또한 멀티모달 LLM을 활용하여 OCR 없이 이미지를 직접 분석하는 방안도 검토할 계획이다.
